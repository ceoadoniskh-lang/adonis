# Як отримати Firebase Token для CI/CD

## Швидка інструкція

1. **Встановіть Firebase CLI** (якщо ще не встановлено):
   ```bash
   npm install -g firebase-tools
   ```

2. **Отримайте токен:**
   ```bash
   firebase login:ci
   ```

3. **Скопіюйте токен**, який з'явиться після команди (виглядає як `1//0abc123...`)

4. **Додайте токен в GitHub:**
   - Перейдіть: `Settings` → `Secrets and variables` → `Actions`
   - Натисніть: `New repository secret`
   - Name: `FIREBASE_TOKEN`
   - Value: вставте скопійований токен
   - Натисніть: `Add secret`

5. **Готово!** Тепер CI/CD буде автоматично деплоїти проект при push в `main`

## Детальніша інструкція

### Що робить команда `firebase login:ci`?

Ця команда:
- Відкриває браузер для авторизації через Google аккаунт
- Після успішної авторизації генерує токен доступу
- Токен дійсний доти, доки ви не вийдете з Firebase CLI (`firebase logout`)

### Безпека

⚠️ **НЕ діліться токеном з іншими!**
- Токен дає повний доступ до вашого Firebase проекту
- Зберігайте його тільки як секрет в GitHub
- Якщо токен потрапив до сторонніх - негайно згенеруйте новий

### Як згенерувати новий токен

Якщо потрібно згенерувати новий токен:
```bash
firebase logout
firebase login:ci
```

Потім оновіть секрет `FIREBASE_TOKEN` в GitHub з новим значенням.
